## 1. 产品目标

AI Notes Dashboard 面向“笔记管理 + 记忆复习”场景，提供以下核心能力：

1. 首页总览学习状态（留存率、待复习、已掌握、近期笔记）。
2. 按智能复习序列执行复习流程，并可提交复习结果。
3. 管理笔记库（浏览、按分类筛选、关键词检索、查看详情）。
4. 新建笔记并可选择是否加入“智能复习队列”。

## 2. 用户与使用场景

1. 学习者每天打开首页查看复习任务。
2. 学习者进入复习列表，选择一条任务进入详情并提交“忘记/模糊/已掌握”。
3. 学习者在笔记库检索主题并查看详细内容。
4. 学习者新建笔记，开启智能复习后将自动生成一条待复习任务。

## 3. 范围定义

### 3.1 In Scope（当前版本）

1. 前端路由与页面渲染。
2. 基于 `localStorage` 的 Mock 数据读写与状态持久化。
3. 复习任务延后、提交结果、笔记创建等核心交互。

### 3.2 Out of Scope（当前版本不包含）

1. 真实后端服务、鉴权、权限体系。
2. 多端同步与冲突处理。
3. 搜索排序策略可配置化。
4. 设置页真实功能（当前为占位入口）。

## 4. 信息架构与路由

1. `/`：首页 Home。
2. `/review`：复习列表页。
3. `/review/detail/:id`：复习详情页。
4. `/review/detail`：复习详情兜底路由（无 id）。
5. `/library`：笔记库页。
6. `/notes/new`：新建笔记页。
7. `/notes/:id`：笔记详情页。
8. `*`：未知路由重定向到 `/`。

## 5. 功能需求与验收标准

### FR-01 首页数据展示

- 描述：进入首页后展示记忆留存率、待复习数、已掌握数、今日复习序列、最近笔记。
- 数据来源：`getHomeData()`。
- 验收标准：
1. 首次进入显示“正在加载首页数据...”。
2. 加载完成后显示关键指标与至少 1 条复习卡片。
3. 点击“查看全部”跳转至 `/review`。
4. 点击最近笔记标题跳转至 `/notes/:id`。

### FR-02 复习列表与延后提醒

- 描述：复习页展示任务统计与任务卡片，支持“开始复习”和“稍后提醒”。
- 数据来源：`getReviewPageData()`，操作 `snoozeReview(id)`。
- 验收标准：
1. 页面加载显示“正在加载复习数据...”。
2. 每条任务包含标题、标签、难度、遗忘临界点。
3. 点击“开始复习”跳转 `/review/detail/:id`。
4. 点击“稍后提醒”按钮进入“处理中...”，完成后该条 `etaText` 更新为“已延后 2h”，`faded=true`。

### FR-03 复习详情与结果提交

- 描述：复习详情展示内容后，用户可提交三种结果。
- 数据来源：`getReviewDetail(id)`，操作 `submitReviewResult(id, result)`。
- 验收标准：
1. 页面加载显示“正在加载复习详情...”。
2. 页面展示标题、标签、核心内容、公式、洞察列表。
3. 三个结果按钮分别为“忘记了/模糊/已掌握”。
4. 提交后跳转 `/review`，且对应任务从复习列表移除。
5. 当结果为 `mastered` 且存在 detail 数据时，笔记库新增一条“复习沉淀”笔记。

### FR-04 笔记库筛选与检索

- 描述：笔记库支持分类切换和关键词搜索。
- 数据来源：`getLibraryData()`。
- 验收标准：
1. 页面加载显示“正在加载笔记库...”。
2. 分类包含“全部”与已有分类。
3. 切换分类后列表仅显示该分类笔记。
4. 搜索匹配范围为标题、摘要、分类（不区分大小写）。
5. 无结果时显示“没有匹配的笔记”。

### FR-05 笔记详情展示

- 描述：从笔记库或首页进入笔记详情，显示完整笔记信息。
- 数据来源：`getNoteDetail(id)`。
- 验收标准：
1. 页面加载显示“正在加载笔记详情...”。
2. 展示标题、日期、分类、记忆度、摘要、正文。
3. 当 `id` 不存在时，降级展示第一条笔记数据（由 Mock 层兜底）。

### FR-06 新建笔记

- 描述：用户可填写标题、正文、分类、智能复习开关并保存。
- 数据来源：`getNewNoteMeta()`，操作 `createNote(payload)`。
- 验收标准：
1. 页面加载显示“正在加载编辑器...”。
2. 分类支持多选；默认读取 meta 的 selected 状态。
3. 点击“保存”后按钮显示“保存中...”，完成后跳转首页。
4. 新建笔记应出现在笔记库列表顶部。
5. 当智能复习开启时，复习列表顶部新增一条对应任务。

## 6. 业务规则

1. 数据持久化键：`localStorage['ai-notes-mock-db-v2']`。
2. 首次启动或解析失败时回填 `seedDB`。
3. `createNote` 默认记忆度 `76`，趋势 `wave`。
4. `submitReviewResult` 会移除复习项；仅 `mastered` 触发“复习沉淀”入库。
5. 复习详情 `id` 缺失或不存在时回退到 `rev-1`。
6. 笔记详情 `id` 缺失或不存在时回退到库中第一条。

## 7. 非功能需求（用于测试门禁）

1. 基础可用性：关键页面首屏可在 3 秒内完成渲染（本地开发环境，mock 数据）。
2. 路由稳定性：非法路径应重定向至 `/`。
3. 持久化一致性：刷新后能保持已创建笔记、已处理复习项状态。
4. 可测试性：关键交互元素需有稳定选择器（建议补充 `data-testid`）。

## 8. 自动化测试需求映射

### 8.1 E2E 主流程（建议 Playwright）

1. `E2E-01` 首页加载与跳转：`/` -> `/review` -> `/review/detail/:id`。
2. `E2E-02` 复习结果流转：提交 `mastered` 后，验证 `/review` 数量减 1 且 `/library` 新增“复习沉淀”类笔记。
3. `E2E-03` 复习延后：点击“稍后提醒”，验证任务文案更新“已延后 2h”。
4. `E2E-04` 新建笔记：在 `/notes/new` 保存后，验证首页/笔记库可见新笔记。
5. `E2E-05` 新建 + 智能复习开关：开启时 `/review` 新增任务，关闭时不新增。
6. `E2E-06` 笔记库筛选检索：分类过滤 + 搜索关键字 + 空结果提示。
7. `E2E-07` 容错路由：访问 `/unknown` 自动回到 `/`。

### 8.2 组件/集成测试（建议 Vitest + RTL）

1. `INT-01` ReviewPage：`handleSnooze` 后触发 reload 并展示最新任务状态。
2. `INT-02` ReviewDetailPage：三种结果按钮禁用逻辑与提交后跳转。
3. `INT-03` NewNoteDetailPage：分类多选、开关状态、payload 组装正确。
4. `INT-04` LibraryPage：`filteredNotes` 在分类与搜索组合下结果正确。

### 8.3 Mock API 单元测试

1. `API-01` `loadDB`：空存储、脏数据、正常数据三类输入。
2. `API-02` `createNote`：空标题/空内容兜底、category 解析、smartReview 分支。
3. `API-03` `submitReviewResult`：不同 result 分支对 reviewItems/libraryNotes 的影响。
4. `API-04` `getNoteDetail`、`getReviewDetail` 的兜底行为。

## 9. 测试数据与环境约束

1. 依赖浏览器 `localStorage`，测试前建议执行清理：`localStorage.removeItem('ai-notes-mock-db-v2')`。
2. 由于 `id` 使用 `Date.now()`，断言时避免写死完整 id，建议按标题或列表变化断言。
3. Mock API 内置 `sleep` 延时，E2E 需使用可见状态等待而非固定 sleep。

## 10. 风险与改进建议（面向自动化测试）

2. 多处 UI 以视觉样式区分状态（如 faded），建议补充语义属性方便断言。
3. `README` 仍是模板内容，建议后续补充项目运行与测试说明。
